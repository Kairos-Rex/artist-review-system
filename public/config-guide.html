配置内置指南

配置已成功内置！

您的API配置现在已经内置在程序中，不再依赖浏览器的localStorage存储。 这意味着配置会永久保存在文件中，即使清除浏览器缓存也不会丢失。

配置方式说明

1 界面配置（推荐）

通过网页界面进行配置，简单易用：

  • • 点击页面右上角的设置按钮
  • • 在API配置选项卡中填写信息
  • • 点击保存配置按钮
  • • 配置会自动保存到文件中

2 直接编辑文件

直接修改HTML文件中的配置变量：

  • • 打开 index.html 文件
  • • 找到 API_CONFIG 变量
  • • 修改 apiKey 和 botId 的值
  • • 保存文件即可生效

文件编辑详细步骤

1 找到配置变量

在 index.html 文件中找到以下代码段（大约在第490行附近）：

// API配置变量 - 直接保存在文件中
let API_CONFIG = {
    apiKey: '', // 在这里填写您的扣子API密钥
    botId: ''   // 在这里填写您的机器人ID
};

2 填写配置信息

将您的API密钥和机器人ID填入对应位置：

let API_CONFIG = {
    apiKey: 'pat_SnrkhHeuv5zIVqz7DLC2sGhfO0z5SxJdAmHuIPoGudo0FbEwwAI6lQnlr9M5bSFX',
    botId: '7548722284312772647'
};

注意：请确保API密钥和机器人ID用引号包围，并且格式正确。

3 保存并测试

完成配置后：

  • • 保存 index.html 文件
  • • 刷新浏览器页面
  • • 在设置面板中测试API连接
  • • 确认配置生效

内置配置的优势

永久保存

配置保存在文件中，不会因清除浏览器缓存而丢失

即开即用

打开页面即可使用，无需重新配置API信息

便于部署

适合服务器部署，配置随代码一起管理

注意事项

  • 安全性：API密钥是敏感信息，请确保文件的访问权限设置合理
  • 格式：编辑时请保持JavaScript语法正确，注意引号和逗号
  • 备份：修改前建议备份原文件，以防配置错误
  • 生效：修改后需要刷新浏览器页面才能生效
